Forms

Standard input

Text input fields allow users on the site to input text, select text, and lookup data via auto-completion. They will usually appear in forms. Users may enter text, numbers, or mixed-format types of input.

Standard text inputs have the class .field-text and must be wrapped in .field-group. The grouping includes the label with .field-label and optional elements such as .field-helper.

example

We can change the appearance and state of a standard input; simply add the .is-error modifier to the input's .field-group container. Javascript toggles the error and places a .field-helper element with the appropriate message.

example

There may be instances where an input must be disabled but still be visible to the user. In such cases, apply a state change of .is-disabled to the .field-group label.

example

Submission input

A single input field's value can be submitted to set a condition, such as a coupon code. Add the module .field-submit containing three buttons representing each state of the field submit module; the states are .field-submit-btn-apply, .field-submit-btn-applied, and .field-submit-btn-remove. Use the class .field-submit-btn to style the button.

The field submit button displays in the apply state when .field-group has the state of .is-filled.

example

Add the .is-success class to the .field-group to enable the applied state. In this state we must utilize disabled on the .field-text input to prevent users from editing the field; users can only remove when in the applied state.

Use a .field-helper element to add a message.

example

On the .is-success state, the user can remove the entered value on hover. The .is-removed class triggers the ability to remove the value on click.

example

Dropdowns

Dropdown menus allow the user to select between multiple selections. The button displays the current state and a down arrow. Upon selection, an overflow menu is displayed over the button and displays the selections.

In most cases, dropdown overflow menus should be limited to 7 to 15 options. If there are more than 15 options, use an autocomplete text field instead. In some select cases, the dropdown overflow and list items in a horizontal layout, allowing for more than 15 options in the overflow.
